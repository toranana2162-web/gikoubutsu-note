**要件定義書**

技工物管理システム

1. プロジェクト概要
背景
歯科医院では、技工物の依頼日・納品予定日・納品状況を紙のノートで管理している。
しかし、手書き管理では記入漏れ、確認漏れ、ページの見落としが発生しやすく、納品チェックに時間がかかるという課題がある。

目的
技工物の依頼・納品状況をデジタルで一元管理し、納品漏れや確認漏れを防ぐ。
誰でも簡単に入力・確認できる仕組みにすることで、現場スタッフの確認作業を効率化する。

スコープ

含むもの

* 技工物情報の登録（印象日、患者名、送るもの（対合・バイト・トレー・参考模型・咬合器・その他から複数チェック）、納品日
* 技工物一覧表示
* 納品予定日・患者名での検索
* 納品済みステータス管理
* 未納品・期限超過の視覚的表示
* スマホ・PC両対応の画面

含まないもの

* 患者個人情報の詳細管理
* 会計・請求機能
* 技工所との外部連携
* ログイン認証機能

2. ユーザーストーリー

As a 歯科医院スタッフ,
I want 技工物の依頼内容を簡単に登録できること,
So that 紙のノートに手書きする手間を減らせる.

As a 歯科医院スタッフ,
I want 納品予定日で技工物を確認できること,
So that 当日届く予定の技工物をすぐに把握できる.

As a 歯科医院スタッフ,
I want 未納品の技工物を一覧で確認できること,
So that 納品チェック漏れを防げる.

As a 歯科医院スタッフ,
I want 患者名や納品日で検索できること,
So that 必要な情報を素早く探せる.

As a 管理者,
I want 納品済み・未納品の状態を管理できること,
So that 技工物の進行状況を正確に把握できる.

F-03　　 技工物一覧表示	登録済みの技工物を一覧で確認できる	高	MVP

F-10	削除機能	不要なデータを削除できる	中	Phase2

4. 非機能要件

* パフォーマンス: ページ読み込みは3秒以内を目標とする
* セキュリティ: 患者情報は最小限の入力にとどめる
* 可用性: 業務中に安定して利用できること
* ユーザビリティ: ITに不慣れなスタッフでも直感的に使える画面にする
* レスポンシブ対応: PC・スマホの両方で利用できること
* 保守性: 後から項目追加やUI修正がしやすい構成にする

5. 制約条件

* 予算: 無料枠で完結
* 期間: 2週間以内
* 技術: Claude Codeを使用して開発
* 想定環境: ブラウザで利用
* デザイン方針: 紙のノートに近い見た目で、現場スタッフが受け入れやすいUIにする

6. 画面遷移

* トップ画面
    * 今日の納品予定
    * 期限超過の未納品
    * 技工物一覧
* 登録画面
    * 印象日
    * 患者名
    * 製作物内容
    * 送るもの
    * 納品予定日
* 一覧画面

    * 検索・絞り込み
    * 納品済みチェック
